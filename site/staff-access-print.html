Ops desk — Bruce (or anyone with PIN)

  1. Open the ops URL above (or Desktop file Hit-A-Lick-Ops-Desk.webloc after running bash scripts/install-live-dashboard-desktop.sh in the repo — file sits on Desktop root, not in a folder).
  2. Enter PIN: default 5505 unless OPS_DASHBOARD_PIN was changed on the server.
  3. If the page is hosted on github.io, paste your API base URL (Cloud Run or Firebase API origin, no trailing slash).
  4. Tab Curator pool → Load pool → check rows → choose lane Bruce Pick's or Giap Pick's → Save to board.
  5. Optional: tab Props (3 days) → pick sport → Load props to verify slate and AI confidence.
  6. Optional: tab Dashboard AI for step-by-step help on the desk.

iOS app — Bruce

  1. Sign in with brucebrian50@gmail.com (Firebase).
  2. Bottom tab Picks — switch Bruce Pick's / Giap Pick's to view boards.
  3. Home → Open Props — ~3-day window; AI confidence on cards; odds on legs.
  4. Account → Curator Studio — open Bruce or Giap lane to publish from the universal pool.

iOS app — Giap

  1. Sign in with giap.social1@gmail.com.
  2. Tab Picks — can view both Bruce and Giap boards (staff entitlement).
  3. Account → Curator Studio — edit Giap lane only (publish Giap’s picks from the pool).
  4. If Bruce shares the ops PIN, Giap may also use the web Curator pool tab (optional).

Subscribers (paying fans)

  1. Subscribe via Stripe (in-app / pricing flow).
  2. Bottom tab Picks shows the Bruce / Giap boards you published.

One-time dev checklist (before picks show)

  1. Firebase Auth: both emails exist with passwords set.
  2. Universal pool has rows (owner: POST /api/curators/pool/add with Bruce Bearer token, or internal tools).
  3. Stripe price IDs configured for curator passes if you sell access.
  4. Deploy API: firebase deploy --only functions:api after secret changes.
Hit-A-Lick is an informational analytics product. It does not accept or place wagers. Passwords in this document are examples—set real values in Firebase and rotate as needed.

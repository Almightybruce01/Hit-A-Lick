Ops desk — Bruce (or anyone with the private PIN)

Unlock with the private ops PIN (same value as Cloud Function env OPS_DASHBOARD_PIN if set).

  1. Open the ops URL above, or double-click Desktop Hit-A-Lick-Ops-Desk.webloc (create with bash scripts/install-live-dashboard-desktop.sh or bash scripts/link-ops-dashboard-desktop.sh — file on Desktop root only).
  2. Enter the private PIN. On github.io, leave API base empty to use production Cloud Run automatically; override only for staging.
  3. Tab Curator pool → Load pool → check rows → lane Bruce Pick's or Giap Pick's → Save to board (replaces upcoming list from pool selection).
  4. Or tab Props (3 days) → sport → Load props → check legs → Append selected to board (adds deduped rows; same hide-after-tipoff rules as the iOS props board).
  5. Optional: tab Dashboard AI for step-by-step help on the desk.

Web — Bruce or Giap (post picks, same API as iOS Curator Studio)

  1. Open https://almightybruce01.github.io/Hit-A-Lick/app.html
  2. Go to Account (bottom tab) → log in on account.html if needed with your Firebase email + password.
  3. On Account, find Curator Studio (web) (only appears for brucebrian50@gmail.com or giap.social1@gmail.com).
  4. Choose Lane (Bruce can pick Bruce or Giap; Giap sees Giap only).
  5. Load universal pool → check rows → Save to board. Subscribers see updates in the app Picks tab.

iOS app — Bruce

  1. Sign in with brucebrian50@gmail.com (Firebase).
  2. Bottom tab Picks — switch Bruce Pick's / Giap Pick's to view boards.
  3. Home → Open Props — ~3-day window; AI confidence on cards; odds on legs.
  4. Account → Curator Studio — open Bruce or Giap lane to publish from the universal pool (same as web Curator Studio).

iOS app — Giap

  1. Sign in with giap.social1@gmail.com.
  2. Tab Picks — can view both Bruce and Giap boards (staff entitlement).
  3. Account → Curator Studio — edit Giap lane only (publish Giap’s picks from the pool).
  4. If Bruce shares the ops PIN, Giap may also use the web Curator pool tab (optional).

Subscribers (paying fans)

  1. Subscribe via Stripe (in-app / pricing flow).
  2. Bottom tab Picks shows the Bruce / Giap boards you published.

One-time dev checklist (before picks show)

  1. Firebase Auth: both emails exist with passwords set.
  2. Universal pool has rows (owner: POST /api/curators/pool/add with Bruce Bearer token, or internal tools).
  3. Stripe price IDs configured for curator passes if you sell access.
  4. Deploy API + hosting: firebase deploy --only functions:api,hosting after secret changes (hosting adds strict headers on ops desk).

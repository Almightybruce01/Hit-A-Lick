Every ops desk feature

        
Use Dashboard AI for step-by-step help. Below is the full map of what this page and the app can do.


        

          
Lock screen (before unlock)

          
    
            
  • PIN — sent as X-Ops-Pin; server default 5505 unless you set OPS_DASHBOARD_PIN.
    • 
            
  • API base URL — required on GitHub Pages: paste your API origin (e.g. Cloud Run https://…run.app or Firebase Hosting with /api rewrite). Same-origin hosting can leave it empty.
    • 
          

        


        

          
Environment

          
    
            
  • Loads GET /api/ops/dashboard: Odds API presence, prop market tier, markets by sport (NBA / NFL / MLB / WNBA), raw JSON, ops heartbeat notes.
    • 
            
  • Use to verify configuration after deploy or secret changes.
    • 
          

        


        

          
All features (this tab)

          
    
            
  • Reference list of tabs and backend routes; share with staff.
    • 
          

        


        

          
Dashboard AI

          
    
            
  • POST /api/ops/dashboard-guide with your question; answers with numbered steps for this desk and curator workflows.
    • 
            
  • With OPENAI_API_KEY on the server: full AI. Without it: static fallback tips (still useful).
    • 
          

        


        

          
AI insights (infrastructure)

          
    
            
  • GET /api/ops/insights — rule-based diagnostics (Odds API quota, Stripe price env vars, curator emails, etc.).
    • 
            
  • Optional OpenAI paragraph summarizing risks (same key as Dashboard AI).
    • 
          

        


        

          
Stripe prices

          
    
            
  • GET /api/billing/pricing-status — which Stripe price IDs are configured (masked). No PIN required for this call on some hosts; desk still useful to view alongside ops.
    • 
          

        


        

          
Curator pool → subscriber boards

          
    
            
  • Load pool — GET /api/ops/universal-pool (Firestore universal pool).
    • 
            
  • Check rows, choose lane Bruce Pick's or Giap Pick's, Save to board — POST /api/ops/curator-board/select.
    • 
            
  • Paid subscribers see these boards in the iOS app Picks tab (bottom bar).
    • 
            
  • Pool rows are created with owner tools (API / scripts), not by subscribers.
    • 
          

        


        

          
iOS app (not on this page)

          
    
            
  • Picks — subscribe / view Bruce & Giap boards.
    • 
            
  • Account → Curator Studio — Bruce (owner) can manage lanes; Giap only the Giap lane.
    • 
            
  • AI Lab — separate from this ops desk; pick/copilot quota for users; unlimited for staff / eligible subs (see app + functions/ai.js).
    • 
            
  • Props & slates — live odds/props; props hide after game start (second-half markets need backend tags).
    •
